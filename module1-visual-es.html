USERNAME se deriva automaticamente de current_user() — no necesitas escribir nada. Cada prompt de Genie Code en este modulo hace referencia a USERNAME, CATALOG, SCHEMA y EXPERIMENT_PATH. Genie Code ve tus celdas abiertas, asi que las detecta automaticamente.

¿Que es Genie Code? Tu asistente de IA dentro del notebook — consciente del contexto desde el inicio. Conoce tus tablas de Unity Catalog, el runtime del cluster y las celdas de codigo abiertas. Piensa en el como un Copilot que realmente entiende tu entorno de datos.

4
Ingenieria de Features
10 min
4

Construye un Pipeline de sklearn que empaquete todo el preprocesamiento dentro del artefacto del modelo. El endpoint de serving aceptara strings crudos ("Maize", "Argentina") — no se necesita codificacion en la app.

Pega en Genie Code
Usando la tabla {CATALOG}.{SCHEMA}.yield_trials y mis variables del notebook
(CATALOG, SCHEMA, USERNAME, EXPERIMENT_PATH):

1. Carga la tabla en un pandas DataFrame. Columna objetivo: "hg/ha_yield".
   Columnas de features: Item, Area, average_rain_fall_mm_per_year, pesticides_tonnes, avg_temp, Year

2. Construye un Pipeline de sklearn que encapsule TODO el preprocesamiento dentro del modelo:
   from sklearn.pipeline import Pipeline
   from sklearn.compose import ColumnTransformer
   from sklearn.preprocessing import OrdinalEncoder

   - ColumnTransformer:
     * OrdinalEncoder en ["Item", "Area"]
       (handle_unknown="use_encoded_value", unknown_value=-1)
     * "passthrough" en ["average_rain_fall_mm_per_year", "pesticides_tonnes", "avg_temp", "Year"]
   - Paso final: XGBRegressor(n_estimators=100, max_depth=5, learning_rate=0.1)

3. Divide los datos 80/20 train/test con random_state=42 (mantener strings crudos en X_train/X_test).
4. Muestra las dimensiones de X_train, X_test, y_train, y_test.
📊 Salida esperada
  • X_train / X_test: DataFrames con columnas de strings crudos (Item="Maize", Area="Argentina")
  • Objeto Pipeline con dos pasos: preprocessor (ColumnTransformer) + regressor (XGBRegressor)
💡 ¿Por que un Pipeline?

El Pipeline de sklearn empaqueta OrdinalEncoder + XGBRegressor en un solo artefacto de modelo. Cuando se registra con mlflow.sklearn.log_model(), el endpoint de serving recibe entradas crudas ("Item": "Maize") y ejecuta la codificacion internamente — la app envia nombres de cultivos, no codigos enteros. No se requiere Feature Store.

R → Python equivalente
recipe() → step_dummy() → workflow() → ColumnTransformer + Pipeline (misma idea)
5
Entrena XGBoost + Registra en MLflow
30 min
5

Entrena el modelo y rastrea todo automaticamente en MLflow. No mas guardar resultados en archivos CSV y perder la pista de cual ejecucion fue cual.

Pega en Genie Code
Entrena el Pipeline de sklearn del Paso 4 y registralo con MLflow estandar
(usa mis variables del notebook: USERNAME, CATALOG, SCHEMA, EXPERIMENT_PATH).

1. Configura el experimento de MLflow a EXPERIMENT_PATH (carpeta del workspace del usuario):
   import mlflow
   mlflow.set_experiment(EXPERIMENT_PATH)

2. Entrena (fit) el pipeline con X_train, y_train.

3. Evalua en X_test:
   - RMSE
   - R²

4. Registra el modelo con mlflow.sklearn.log_model dentro de un run:
   with mlflow.start_run():
       mlflow.log_params({"n_estimators": 100, "max_depth": 5, "learning_rate": 0.1})
       mlflow.log_metrics({"rmse": rmse, "r2": r2})
       mlflow.sklearn.log_model(
           sk_model        = pipeline,
           artifact_path   = "gdm_yield_pipeline",
           registered_model_name = f"{CATALOG}.{SCHEMA}.gdm-yield-model-{USERNAME}"
       )
   Esto registra el pipeline completo (OrdinalEncoder + XGBoost) como un solo artefacto.
   El endpoint de serving recibira strings crudos y ejecutara la codificacion internamente.

5. Crea dos graficos de plotly:
   a. Grafico de barras horizontales con las importancias de features extraidas del pipeline:
        pipeline.named_steps["regressor"].feature_importances_
      Nombres de display de features (en orden del ColumnTransformer):
        ["Tipo de Cultivo", "Pais", "Lluvia (mm/ano)", "Pesticidas (t)", "Temp Prom (°C)", "Ano"]
      Ordenar de mayor a menor. Mostrar las 6 barras.
   b. Grafico de dispersion: rendimiento predicho vs real en X_test,
      con una linea diagonal de referencia (linea de prediccion perfecta).
Muestra ambos graficos en linea.
📊 Salida esperada
  • RMSE: aproximadamente 20,000 – 40,000 hg/ha
  • R²: aproximadamente 0.85 – 0.95
  • Ejecucion de MLflow registrada en {EXPERIMENT_PATH} (tu carpeta del workspace)
  • Modelo registrado en UC: {CATALOG}.{SCHEMA}.gdm-yield-model-{USERNAME}
  • El artefacto del modelo es el pipeline completo — OrdinalEncoder + XGBoost empaquetados juntos
  • Grafico de importancia de features muestra 6 barras limpias ("Crop Type", "Country", etc.)

6
Despliega el Endpoint de Model Serving
30 min
6

Convierte tu modelo registrado en una API REST en vivo. No hay infraestructura que administrar — Databricks maneja todo de forma serverless.

  1. 1Barra lateral izquierda → Machine Learning → Models
  2. 2Busca gdm-yield-model-{USERNAME} (tu modelo registrado del Paso 5) → clic en Serve this model
  3. 3Nombre del endpoint: gdm-yield-{USERNAME}
  4. 4Computo: selecciona Serverless (recomendado)
  5. 5Clic en Create → espera 2–3 minutos hasta que el estado sea: Ready
Prueba el endpoint

Una vez que el estado sea Ready, pega esto en la pestana Query. Como el pipeline esta empaquetado dentro del modelo, el endpoint acepta strings crudos — no se necesita pre-codificacion.

    "Year": 2020
  }]
}
