import re

print(f"📍 USERNAME       : {USERNAME}")
print(f"📍 Your namespace : {CATALOG}.{SCHEMA}")

📌 Second cell — create your personal schema
# Create your personal schema (run once)
spark.sql(f"CREATE SCHEMA IF NOT EXISTS {CATALOG}.{SCHEMA}")
print(f"✅ Schema ready: {CATALOG}.{SCHEMA}")
💡 Why env vars?

USERNAME is derived automatically from current_user() — no typing required. Every Genie Code prompt in this module references USERNAME, CATALOG, SCHEMA, and EXPERIMENT_PATH. Genie Code sees your open cells, so it picks them all up automatically.

What is Genie Code? Your in-notebook AI assistant — context-aware from the start. It knows your Unity Catalog tables, cluster runtime, and open code cells. Think of it as a Copilot that actually understands your data environment.

4
Feature Engineering
10 min
4

Build a sklearn Pipeline that bundles all preprocessing inside the model artifact. The serving endpoint will accept raw strings ("Maize", "Argentina") — no encoding needed in the app.

Paste into Genie Code
Using the table {CATALOG}.{SCHEMA}.yield_trials and my notebook variables
(CATALOG, SCHEMA, USERNAME, EXPERIMENT_PATH):

1. Load the table into a pandas DataFrame. Target column: "hg/ha_yield".
   Feature columns: Item, Area, average_rain_fall_mm_per_year, pesticides_tonnes, avg_temp, Year

2. Build a sklearn Pipeline that encapsulates ALL preprocessing inside the model:
   from sklearn.pipeline import Pipeline
   from sklearn.compose import ColumnTransformer
   from sklearn.preprocessing import OrdinalEncoder

   - ColumnTransformer:
     * OrdinalEncoder on ["Item", "Area"]
       (handle_unknown="use_encoded_value", unknown_value=-1)
     * "passthrough" on ["average_rain_fall_mm_per_year", "pesticides_tonnes", "avg_temp", "Year"]
   - Final step: XGBRegressor(n_estimators=100, max_depth=5, learning_rate=0.1)

3. Split the data 80/20 train/test with random_state=42 (keep raw strings in X_train/X_test).
4. Show the shapes of X_train, X_test, y_train, y_test.
📊 Expected output
  • X_train / X_test: DataFrames with raw string columns (Item="Maize", Area="Argentina")
  • Pipeline object with two steps: preprocessor (ColumnTransformer) + regressor (XGBRegressor)
💡 Why a Pipeline?

The sklearn Pipeline bundles OrdinalEncoder + XGBRegressor into a single model artifact. When logged with mlflow.sklearn.log_model(), the serving endpoint receives raw inputs ("Item": "Maize") and runs encoding internally — the app sends crop names, not integer codes. No Feature Store required.

R → Python parallel
recipe() → step_dummy() → workflow() → ColumnTransformer + Pipeline (same idea)

Deploy Model Serving Endpoint
30 min
6

Turn your registered model into a live REST API. No infrastructure to manage — Databricks handles everything serverlessly.

  1. 1Left sidebar → Machine Learning → Models
  2. 2Find gdm-yield-model-{USERNAME} (your registered model from Step 5) → click Serve this model
  3. 3Endpoint name: gdm-yield-{USERNAME}
  4. 4Compute: select Serverless (recommended)
  5. 5Click Create → wait 2–3 minutes for status: Ready
Test the endpoint

Once status is Ready, paste this in the Query tab. Because the pipeline is packaged inside the model, the endpoint accepts raw strings — no pre-encoding needed.

{
  "inputs": [{
    "Item": "Maize",
    "Area": "Argentina",
    "average_rain_fall_mm_per_year": 750.0,
    "pesticides_tonnes": 32000.0,
    "avg_temp": 18.5,
    "Year": 2020
  }]
}
📊 Expected response
  • Endpoint status: Ready
  • Response: {"predictions": [45320.5]} — yield in hg/ha ≈ 4.5 t/ha
  • Payload uses raw strings ("Maize", "Argentina") — the pipeline handles encoding internally
  • Endpoint URL format: https://<workspace>.azuredatabricks.net/serving-endpoints/gdm-yield-{USERNAME}/invocations
